Образ, который создает Docker, на самом деле не является специфичным для Docker образом — это образ OCI ( Open Container Initiative ). Любой образ, совместимый с OCI, независимо от инструмента, который вы используете для его создания, будет выглядеть одинаково для Kubernetes.
В свою очередь kind — это инструмент для запуска локальных кластеров Kubernetes с использованием «узлов» контейнера Docker. kind был в первую очередь разработан для тестирования самого Kubernetes, но может использоваться для локальной разработки или CI.
